Capabilities

Everything your agents
need to work as one.

Core

Queen-led orchestration

A Queen drone writes the plan, assigns tasks to the right agents, and reviews their output — you set the goal, agents handle execution.

Architecture

Persistent colonies

Named, configurable colonies with a visual org graph (Comb). Drones run in parallel or sequence — the relationship graph is your workflow.

Security

Builder privilege model

Least-privilege by design. Only Builder drones get filesystem and command access. Planners, reviewers, and PMs stay sandboxed — no accidental writes.

Workflow

Chained task pipelines

Connect agent outputs into multi-step pipelines with delegates-to, collaborates-with, and reviews-for relationships. What takes hours becomes one run.

How it works

Three steps.
Any task.

01

Define your colony

Give each drone a name, role, persona, and set of tools. Connect them in the Comb graph. Designate Builders with filesystem access — everyone else stays sandboxed.

02

Hit play on an issue

Assign a Linear issue and press Play. The Queen reads the task, writes a plan, and dispatches work to the right drones — in parallel or sequence based on your Comb graph.

03

Review and iterate

Watch the activity feed in real time. Each drone posts output as a Linear comment. The Queen reviews, approves, or loops back. Examine, export, ship.
