Everything a colony needs

Orchestration

Queen-driven coordination

One orchestrator agent reads your goal and assigns specialized workers — no manual wiring required. Describe the outcome; the colony figures out the path.

Context

Shared workspace memory

Every agent in a colony reads and writes to a shared workspace. Context flows automatically between tasks — no copy-pasting, no lost state.

Interface

macOS-native controls

Launch, pause, and inspect colonies from the menu bar. Full output streaming, live agent status, no browser required. Built for the Mac, not ported to it.

Speed

Parallel execution

Workers run concurrently. Complex multi-step workflows finish in the time it takes to complete one task sequentially. Parallelism without orchestration overhead.

How it works

A colony that thinks

You describe a goal. The Queen agent breaks it into tasks and dispatches specialized Workers — each focused, each fast. Results flow back automatically, assembled into the output you asked for.

No pipelines to configure. No chains to debug. Just a goal and a colony that gets it done.

hive run colony.yaml 
# colony.yaml — research and summarize task
name: research-brief

queen:
  model: claude-opus-4-5
  goal: >
    Research the competitive landscape for AI agent
    frameworks and produce a two-page executive brief.

workers:
  - name: researcher
    model: claude-sonnet-4-6
    tools: [web_search, read_file]
  - name: writer
    model: claude-sonnet-4-6
    tools: [write_file]
